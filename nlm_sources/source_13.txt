# Epistory - コード分析レポート
## 基本情報
- リポジトリ: chuto-pika/Epistory
- 作者: chuto-pika
- フレームワーク: Ruby on Rails 7.0 + Tailwind CSS + OpenAI API
- アーキテクチャ: MVC + Multi-Step Wizard + AI Message Generation + Session-based Draft
- 規模: 中規模（ファイル数: 1901）
- 複雑度スコア: 4/5
- 使用言語: Ruby, HTML, Python, JavaScript, CSS, Shell, Dockerfile

## アプリ概要
感謝メッセージ生成アプリ。6ステップのウィザード形式で相手・場面・気持ちなどを入力し、OpenAI APIでパーソナライズされた感謝メッセージを自動生成する。

## 設計の鍵（Design Keys）
1. 6ステップウィザードパターン（session-based draft管理）
2. OpenAI APIによるメッセージ生成 + パート別再生成 + AIブラッシュアップ
3. ログイン不要でも1通作成可能なセッションベース認可

## 学び（Learnings）
1. マルチステップウィザードのsession管理とConcernによるモジュール化
2. OpenAI API連携（ruby-openai gem）によるAIテキスト生成
3. 日次制限付きAI機能（ai_refine_daily_used + usage_date）の設計

## BUD分析
- ボトルネック数: 1
- 不必要な作業: 2
- 重複する作業: 1

## セキュリティ懸念
1. セッションベースのメッセージ所有権チェック（session[:created_message_id]）は改ざんリスクあり
2. OpenAI APIキーの管理方法（credentials経由か環境変数か）を確認すべき
3. ファイル数が1901と多く、不要ファイル（テスト生成物やビルドキャッシュ）が含まれている可能性
