# iStanding - コード分析レポート
## 基本情報
- リポジトリ: git-kaz/istanding
- 作者: git-kaz
- フレームワーク: Ruby on Rails 8.1 + Hotwire
- アーキテクチャ: MVC + Solid Queue + Web Push
- 規模: 中規模（ファイル数: 214）
- 複雑度スコア: 3/5
- 使用言語: HTML, Ruby, JavaScript, Dockerfile, Shell, CSS

## アプリ概要
座りすぎ防止のための姿勢・運動管理アプリ。座位時間をタイマーで計測し、一定時間経過でWeb Push通知を送信、ストレッチ運動を提案する。

## 設計の鍵（Design Keys）
1. Solid Queue/Solid Cache/Solid Cableによるインフラ統合（Rails 8方式）
2. Web Push通知によるリアルタイムアラート機能
3. Chartkick + Groupdateによる座位時間・運動回数の可視化

## 学び（Learnings）
1. Rails 8のSolid Queue/Cache/Cableの統合アーキテクチャ
2. Web Push APIによるブラウザプッシュ通知の実装
3. Devise + OmniAuth（Google OAuth2）による認証基盤

## BUD分析
- ボトルネック数: 1
- 不必要な作業: 1
- 重複する作業: 1

## セキュリティ懸念
1. ゲストユーザーのパスワードがSecureRandom生成だが、emailが固定値（guest@example.com）
2. Web Push認証キー（p256dh, auth）がユーザーテーブルに直接保存されている
3. subscribeアクションでStrong Parametersが十分に適用されていない
