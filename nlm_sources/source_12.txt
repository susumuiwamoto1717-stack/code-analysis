# Okan - コード分析レポート
## 基本情報
- リポジトリ: saku-56/OKAN
- 作者: saku-56
- フレームワーク: Ruby on Rails 7.2 + Tailwind CSS 4 + Hotwire
- アーキテクチャ: MVC + Form Object + Service Object + LINE Bot Integration
- 規模: 中規模（ファイル数: 276）
- 複雑度スコア: 4/5
- 使用言語: Ruby, HTML, JavaScript, CSS, Dockerfile, Shell

## アプリ概要
薬の在庫管理・通院スケジュール管理アプリ。処方薬の在庫を自動計算し、在庫切れ前にLINE通知を送信。通院予定日のリマインダー機能も備える。

## 設計の鍵（Design Keys）
1. Form Objectパターン（UserMedicineForm）による複雑なフォームバリデーション
2. Service Objectパターン（MedicineNotificationService, ConsultationNotificationService）
3. LINE Bot API v2によるプッシュ通知 + wheneverによるcron連携

## 学び（Learnings）
1. Form ObjectとService Objectによるファットモデル/コントローラーの回避
2. LINE Bot API v2を使ったプッシュメッセージ通知の実装
3. GitHub Actions + wheneverによる定期タスク実行（在庫消費・通知）

## BUD分析
- ボトルネック数: 1
- 不必要な作業: 1
- 重複する作業: 1

## セキュリティ懸念
1. LINE Bot APIのエラーハンドリングでrescue => eが広すぎる（例外を握りつぶしている）
2. credentials/がenvironment別に分離されているのは良い設計
3. OmniAuthでLINEログイン時にemail未取得の場合にダミーメールを生成している
